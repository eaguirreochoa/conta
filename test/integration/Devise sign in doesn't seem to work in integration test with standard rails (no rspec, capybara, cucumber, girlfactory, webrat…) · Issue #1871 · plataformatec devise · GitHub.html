Devise integration signin

Devise sign in doesn't seem to work in integration test with standard rails (no rspec, capybara, cucumber, girlfactory, webrat…)

I used the following fixtures

administrator:
email: 'Alan.Admin@example.org'
encrypted_password: <%= User.new.send(:password_digest, 'good_adm_PW') %>

tester:
email: 'Tigran.Test@example.org'
encrypted_password: <%= User.new.send(:password_digest, 'good_test_PW') %>

and the following integration test

require 'test_helper'

class UserAdministratorTest < ActionDispatch::IntegrationTest
fixtures :all

def setup
@adm = users(:administrator)
@adm_pw = 'good_adm_PW'
end

test "'good_adm_PW' should be a valid administrator password" do
assert @adm.valid_password?(@adm_pw), "'#{@adm_pw}' is not a valid administrator password"
end

test "'bad_adm_PW' should not be a valid administrator password" do
assert !@adm.valid_password?('bad_adm_PW'), "'bad_adm_PW' is a valid administrator password"
end

test "administator should not sign in with bad password" do
post user_session_path, :user => { :email => @adm.email, :password => 'bad_adm_PW'}
assert_equal flash[:alert], 'Invalid email or password.', 'Administrator authentication succeed with a bad password.'
end

test "administrator should sign in with a good password" do
post_via_redirect user_session_path, :user => { :email => @adm.email, :password => @adm_pw}
assert_equal flash[:notice], 'Signed in successfully.', 'Administrator authentication failed with a good password.'
end

end

Tests from 1 to 3 succeed but test 4 fails.

josevalim commented on 24 May 2012

The issue is happening because your fixture uses 'Alan.Admin@example.org' as e-mail but Devise normalizes all e-mails to be lowercase. Then it cannot find the user because it expects the e-mail to be lowercase. You can either use factories to abstract creating the fixtures for you, disable the normalization or fix your fixtures.

nortcele commented on 24 May 2012

Yes, fixing my fixtures make the tests work fine. Thank you a lot. Sorry to made you waste time.
